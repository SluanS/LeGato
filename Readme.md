# LeGato
## Minha primeira Landing page feita em HTML, CSS e JavaScript.

### Descrição:
Projeto realizado através de proposta estabelecida em sala de aula: Uma landing page totalmente funcional de um cliente de PetShop fictício.

### Idealização:
#### Protótipo feito em figma:
<div>
  <img height="350" alt="image" src="https://github.com/user-attachments/assets/f4299246-8278-4c27-bf0e-09650021709d">

Como uma imersão prática nos cenários do mercado de trabalho, fui provocado a idealizar a criação de uma página web para um cliente ficticio desde o seu ínicio, onde simulei as etapas desde o levantamento de requisitos até a prototipação inicial do design da página em figma.

A seguir, mostrarei algumas caracteristicas da aplicação de acordo com os seguintes requisitos base: 
- Layout responsivo
- Interatividade básica com JavaScript (Captura de cookies, carrossel de imagens e menu hamburger)
- Estilização com CSS

### Psicologia das cores
Ao aplicar conceitos da teoria das cores, criei uma interface com cores suaves e convidativas, visando um efeito de cuidado (uma caracteristica pilar de qualquer serviço de PetShop) e confiança (Uma importante impressão para a prospecção de novos leads).
Com tons váriados de azul, cinza e roxo, a página apresenta os diversos serviços, contatos e descrições do nosso cliente.

### Interatividade com JavaScript

Apliquei JavaScript para criar um dinamismo básico através de interações diretas com a DOM, além do já mencionado menu responsivo (que possui em seu funcionamento uma base de JavaScript) e do carrossel de imagens(que troca a imagem exibida a cada periodo de tempo), apliquei a linguagem em diversos outros campos da DOM. 
<br>
Com esse projeto, pude ter a minha primeira experiência prática com captura de cookies, criando um card específico para captar as preferências do usuário:

<img width="824" height="285" alt="image" src="https://github.com/user-attachments/assets/cf939525-de50-4a32-9d15-924f57e6b297" />

Quando o usuário aceitar os a captura de cookie, essa informação por sí só será armazenada como um cookie, essa informação será expirada dentro de um ano a partir da data de criação do cookie. 
<br>
Caso o usuário não aceite a captura de cookie, o card seguirá a ser exibidos em proximas visitas ao site.
